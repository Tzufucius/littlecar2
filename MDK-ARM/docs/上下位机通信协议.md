# STM32F407 上下位机通信协议

## 1. 定位

本文档定义 Jetson / Windows 上位机与 STM32F407 下位机之间的主控通信协议。

通信目标：

- 上位机通过 `USART6` 向 STM32 下发控制命令。
- STM32 解析命令后调用内部业务接口执行动作。
- STM32 通过 ACK、查询响应和周期状态向上位机反馈结果。
- 协议保持简单稳定，同时保留后续扩展传感器能力的空间。

本协议不是电机、舵机、OPS、WIT 等设备的原始协议。上位机只面对统一命令，STM32 内部再分发到 `advance_chassis`、`advance_world`、`drive_bus_servo`、`sensor_ops`、`sensor_wit` 等模块。

能力清单维护在：

```text
docs/上位机可调用能力清单.md
```

## 2. 链路参数

主通信口使用 `USART6`。

| 项目 | 配置 |
| --- | --- |
| STM32 TX | `PC6 / USART6_TX` |
| STM32 RX | `PC7 / USART6_RX` |
| 波特率 | `115200` |
| 数据位 | `8` |
| 停止位 | `1` |
| 校验 | `None` |
| 流控 | `None` |
| 接收方式 | `DMA Circular + UART IDLE` |
| 发送方式 | 普通发送或 DMA Normal |

硬件要求：

- 上位机 TX 接 STM32 `PC7 / USART6_RX`。
- 上位机 RX 接 STM32 `PC6 / USART6_TX`。
- 必须共地。
- 双方电平均需匹配，不能把 RS-232 电平直接接到 STM32 3.3V UART。

当前工程已有 `host_rx` 原始接收调试模块。正式协议实现时，`USART6` 应由协议层接管，避免 `host_rx` 和正式协议层同时启动同一个串口 DMA 接收。

## 2.1 world 坐标系约定

STM32 内部通过 `advance_world` 维护工程统一坐标系。OPS 原始坐标只作为输入，不直接作为最终任务坐标。

约定如下：

- `world +Y`：小车上电/校准时的车头方向。
- `world +X`：小车上电/校准时的右侧方向。
- `yaw = 0 deg`：车头朝向 `world +Y`。
- yaw 逆时针为正。
- 协议中位置使用 `int32_t mm`，角度使用 `int32_t cdeg`。

OPS 静止初始化完成后，STM32 调用 `AdvanceWorld_ResetOrigin()`，将当前 OPS `x/y/yaw` 记录为 world 原点。后续 `SENSOR_GET_POSE_FUSION` 和 `MSG_STATUS` 中的位姿字段应来自 `advance_world`，不是 OPS 原始值。

## 3. 基础帧格式

协议使用固定二进制帧：

```text
Header1 Header2 Version MsgType CmdSet CmdID Seq_L Seq_H Length Payload CRC_L CRC_H
```

| 偏移 | 字段 | 长度 | 类型 | 说明 |
| ---: | --- | ---: | --- | --- |
| 0 | Header1 | 1 | `uint8_t` | 固定 `0x5A` |
| 1 | Header2 | 1 | `uint8_t` | 固定 `0xA5` |
| 2 | Version | 1 | `uint8_t` | 当前 `0x01` |
| 3 | MsgType | 1 | `uint8_t` | 消息类型 |
| 4 | CmdSet | 1 | `uint8_t` | 命令组 |
| 5 | CmdID | 1 | `uint8_t` | 命令 ID |
| 6 | Seq_L | 1 | `uint8_t` | 序号低字节 |
| 7 | Seq_H | 1 | `uint8_t` | 序号高字节 |
| 8 | Length | 1 | `uint8_t` | Payload 长度，`0~255` |
| 9... | Payload | N | bytes | 数据区 |
| 9+N | CRC_L | 1 | `uint8_t` | CRC16 低字节 |
| 10+N | CRC_H | 1 | `uint8_t` | CRC16 高字节 |

总长度：

```text
total_len = 9 + Length + 2
```

首版只使用上述固定字段。命令行为由 `MsgType + CmdSet + CmdID` 决定。

## 4. CRC 与编码

CRC 使用 `CRC16-Modbus`：

| 项目 | 值 |
| --- | --- |
| 初值 | `0xFFFF` |
| 多项式 | `0xA001` |
| 输出顺序 | 低字节在前 |
| 计算范围 | `Version` 到 `Payload` 最后一个字节 |

计算表达式：

```c
crc = CRC16_Modbus(&frame[2], 7 + payload_len);
```

不参与 CRC 的字段：

- `Header1`
- `Header2`
- `CRC_L`
- `CRC_H`

多字节字段统一小端：

| 类型 | 编码 |
| --- | --- |
| `uint16_t` / `int16_t` | 低字节在前 |
| `uint32_t` / `int32_t` | 低字节在前 |

协议 Payload 不直接传 `float`。物理量转为定点整数：

| 物理量 | 类型 | 单位 |
| --- | --- | --- |
| 电机转速 | `int16_t` | RPM |
| 位置 | `int32_t` | mm |
| 角度 | `int32_t` | 0.01 deg |
| 角速度 | `int16_t` | 0.01 deg/s |
| 加速度 | `int16_t` | 0.001 g |
| 时间 | `uint32_t` | ms |

## 5. 消息类型

```c
typedef enum
{
    MSG_CMD    = 0x01,
    MSG_ACK    = 0x02,
    MSG_DATA   = 0x03,
    MSG_STATUS = 0x04
} ProtoMsgType_t;
```

| MsgType | 方向 | 用途 |
| ---: | --- | --- |
| `0x01` | 上位机 -> STM32 | 控制命令或查询命令 |
| `0x02` | STM32 -> 上位机 | 命令接收和执行结果 |
| `0x03` | STM32 -> 上位机 | 查询响应数据 |
| `0x04` | STM32 -> 上位机 | 周期状态上报 |

规则：

- 所有有效 `MSG_CMD` 都返回 `MSG_ACK`。
- 查询类命令先返回 `MSG_ACK`，再返回同 `Seq` 的 `MSG_DATA`。
- 周期状态统一使用 `MSG_STATUS`。
- CRC 错误帧直接丢弃；如果无法可信读取 `Seq`，不强制回复。

## 6. 命令组

```c
typedef enum
{
    CMDSET_SYSTEM  = 0x01,
    CMDSET_SAFETY  = 0x02,
    CMDSET_CHASSIS = 0x03,
    CMDSET_SERVO   = 0x04,
    CMDSET_SENSOR  = 0x05,
    CMDSET_CONFIG  = 0x06,
    CMDSET_DEBUG   = 0x7F
} ProtoCmdSet_t;
```

| CmdSet | 用途 |
| ---: | --- |
| `0x01` | 系统通信、心跳、模式 |
| `0x02` | 急停、安全停止、恢复 |
| `0x03` | 底盘控制 |
| `0x04` | 舵机、夹爪、机械臂 |
| `0x05` | 传感器查询和状态上报 |
| `0x06` | 参数、预设点位、预设姿态 |
| `0x7F` | 调试命令 |

新增传感器优先放在 `CMDSET_SENSOR` 中，通过新增 `CmdID` 扩展，不修改基础帧格式。

## 7. ACK

ACK Payload 固定 4 字节：

```text
ack_seq_l ack_seq_h result detail
```

| 偏移 | 字段 | 类型 | 长度 | 说明 |
| ---: | --- | --- | ---: | --- |
| 0 | ack_seq | `uint16_t` | 2 | 被确认的上位机命令序号 |
| 2 | result | `uint8_t` | 1 | 执行结果 |
| 3 | detail | `uint8_t` | 1 | 细节码，首版可填 0 |

`result` 定义：

| 值 | 名称 | 含义 |
| ---: | --- | --- |
| `0x00` | `OK` | 执行成功或命令已接受 |
| `0x01` | `BAD_CRC` | CRC 错误 |
| `0x02` | `BAD_LENGTH` | Payload 长度错误 |
| `0x03` | `BAD_PARAM` | 参数错误 |
| `0x04` | `BUSY` | 系统忙 |
| `0x05` | `DENIED` | 当前状态不允许执行 |
| `0x06` | `UNKNOWN_CMD` | 未知命令 |
| `0x07` | `FAULT` | 系统故障 |

ACK 帧字段：

| 字段 | 取值 |
| --- | --- |
| `MsgType` | `MSG_ACK` |
| `CmdSet` | 原命令 `CmdSet` |
| `CmdID` | 原命令 `CmdID` |
| `Seq` | STM32 发送序号或原命令 `Seq`，实现时需固定一种 |
| `Payload.ack_seq` | 原命令 `Seq` |

建议 STM32 首版直接让 ACK 帧 `Seq` 等于原命令 `Seq`，上位机匹配逻辑更简单。

## 8. 首版命令

首版只实现能稳定控制底盘的最小集。

### 8.1 SYSTEM

| CmdID | 命令 | Payload | 说明 |
| ---: | --- | --- | --- |
| `0x01` | `SYS_PING` | 无 | 通信测试 |
| `0x02` | `SYS_HEARTBEAT` | `uint32_t jetson_time_ms` | 上位机心跳 |
| `0x03` | `SYS_SET_MODE` | `uint8_t mode` | 设置控制模式 |

心跳建议 50~100 ms 发送一次。STM32 超过 300 ms 未收到心跳时，应停止底盘并进入安全状态。

### 8.2 SAFETY

| CmdID | 命令 | Payload | 说明 |
| ---: | --- | --- | --- |
| `0x01` | `SAFETY_ESTOP` | `uint8_t source` | 立即急停 |
| `0x02` | `SAFETY_SAFE_STOP` | `uint8_t mode` | 安全停止 |
| `0x03` | `SAFETY_CLEAR` | 无 | 清除可恢复安全状态 |

首版安全控制只要求停止底盘。后续机械臂、舵机等安全动作再接入同一命令组。

### 8.3 CHASSIS

底盘命令对齐 `Core/Inc/chassis_motion.h`。

| CmdID | 命令 | Payload 长度 | STM32 内部落点 |
| ---: | --- | ---: | --- |
| `0x01` | `CHASSIS_ENABLE` | 1 | `Chassis_Enable()` |
| `0x02` | `CHASSIS_STOP` | 1 | `Chassis_Stop()` / `Chassis_SmoothStop()` |
| `0x03` | `CHASSIS_SET_MOTOR_RPM` | 9 | `Chassis_SetMotorRPMEx()` |
| `0x04` | `CHASSIS_MOVE_MECANUM` | 7 | `Chassis_MoveMecanumEx()` |

`CHASSIS_ENABLE`：

| 偏移 | 字段 | 类型 | 说明 |
| ---: | --- | --- | --- |
| 0 | enable | `uint8_t` | `0` 失能，`1` 使能 |

`CHASSIS_STOP`：

| 偏移 | 字段 | 类型 | 说明 |
| ---: | --- | --- | --- |
| 0 | mode | `uint8_t` | `0` 平滑停止，`1` 立即停止 |

`CHASSIS_SET_MOTOR_RPM`：

| 偏移 | 字段 | 类型 | 单位 |
| ---: | --- | --- | --- |
| 0 | lf_rpm | `int16_t` | RPM |
| 2 | rf_rpm | `int16_t` | RPM |
| 4 | lr_rpm | `int16_t` | RPM |
| 6 | rr_rpm | `int16_t` | RPM |
| 8 | acc | `uint8_t` | Emm 加速度参数 |

`CHASSIS_MOVE_MECANUM`：

| 偏移 | 字段 | 类型 | 单位 |
| ---: | --- | --- | --- |
| 0 | forward_rpm | `int16_t` | RPM |
| 2 | strafe_rpm | `int16_t` | RPM |
| 4 | rotate_rpm | `int16_t` | RPM |
| 6 | acc | `uint8_t` | Emm 加速度参数 |

方向约定：

- `forward_rpm > 0` 表示前进。
- `strafe_rpm > 0` 表示右平移。
- `rotate_rpm > 0` 表示右旋转。

## 9. 传感器扩展

传感器统一放在 `CMDSET_SENSOR`。首版可先不实现查询，后续按此规则增加。

### 9.1 查询命令

| CmdID | 命令 | Payload | 返回 |
| ---: | --- | --- | --- |
| `0x01` | `SENSOR_GET_OPS` | 无 | `MSG_DATA` 返回 OPS 位姿 |
| `0x02` | `SENSOR_GET_IMU` | 无 | `MSG_DATA` 返回 IMU 数据 |
| `0x03` | `SENSOR_GET_POSE_FUSION` | 无 | 预留融合位姿 |
| `0x04` | `SENSOR_GET_MOTOR_STATE` | 可选电机 ID | 预留电机状态 |
| `0x80` | `SENSOR_STATUS` | 无 | 固定用于周期状态上报 |

`MSG_DATA` 归属规则：

| 字段 | 取值 |
| --- | --- |
| `MsgType` | `MSG_DATA` |
| `CmdSet` | 原查询命令 `CmdSet` |
| `CmdID` | 原查询命令 `CmdID` |
| `Seq` | 原查询命令 `Seq` |
| `Payload` | 查询结果 |

### 9.2 新增传感器规则

新增传感器时，只做以下扩展：

1. 在 `CMDSET_SENSOR` 中分配新的 `CmdID`。
2. 定义查询 Payload 和 `MSG_DATA` 返回 Payload。
3. 明确每个字段的类型、单位、缩放倍数和有效位。
4. 如需周期状态，只在 `MSG_STATUS` 中放关键摘要，不把完整传感器数据塞进状态帧。
5. 保持传感器原始协议在 STM32 内部模块中解析，不透传给上位机。

建议 `CmdID` 分配：

| 范围 | 用途 |
| ---: | --- |
| `0x01~0x1F` | 常用传感器查询 |
| `0x20~0x3F` | 融合数据、估计状态 |
| `0x40~0x5F` | 执行器或电机状态查询 |
| `0x80` | 周期状态上报 |
| `0xE0~0xEF` | 临时调试传感器，稳定后再转正 |

## 10. 周期状态

STM32 可按 10 Hz 发送 `MSG_STATUS`。状态帧只放高频控制需要的摘要，不替代传感器查询。

`MSG_STATUS` 固定字段：

| 字段 | 取值 |
| --- | --- |
| `MsgType` | `MSG_STATUS` |
| `CmdSet` | `CMDSET_SENSOR` |
| `CmdID` | `0x80` |

建议 Payload：

| 偏移 | 字段 | 类型 | 说明 |
| ---: | --- | --- | --- |
| 0 | stm32_time_ms | `uint32_t` | HAL tick |
| 4 | system_state | `uint8_t` | 系统状态 bit mask |
| 5 | fault_code | `uint8_t` | 故障码 |
| 6 | chassis_state | `uint8_t` | 底盘状态 |
| 7 | sensor_valid_mask | `uint8_t` | 传感器有效位 |
| 8 | active_seq | `uint16_t` | 当前命令，无则 0 |
| 10 | last_done_seq | `uint16_t` | 最近完成命令，无则 0 |
| 12 | pos_x_mm | `int32_t` | 当前位置 X，可无效 |
| 16 | pos_y_mm | `int32_t` | 当前位置 Y，可无效 |
| 20 | yaw_cdeg | `int32_t` | 航向角，可无效 |

`sensor_valid_mask` 建议：

| bit | 含义 |
| ---: | --- |
| 0 | OPS 有效 |
| 1 | IMU 加速度有效 |
| 2 | IMU 角速度有效 |
| 3 | IMU 姿态角有效 |
| 4~7 | 预留给新增传感器摘要 |

## 11. STM32 解析与执行规则

STM32 侧建议分层：

```text
USART6 DMA + IDLE
  ↓
协议字节流解析
  ↓
完整帧校验
  ↓
命令分发
  ↓
advance_chassis / advance_world / drive_bus_servo / sensor_ops / sensor_wit
```

规则：

- 中断或 DMA 回调中只搬运数据，不直接控制电机或舵机。
- 协议层负责找帧头、读 Length、校验 CRC、投递完整帧。
- 命令执行放在主循环或命令队列中完成。
- 每个 `CmdID` 必须检查 Payload 长度。
- 未知命令返回 `ACK_UNKNOWN_CMD`。
- 参数越界返回 `ACK_BAD_PARAM`，不得执行部分动作。
- 心跳超时、急停、故障状态应阻止普通运动命令。

## 12. 上位机发送规则

上位机规则：

- 每条 `MSG_CMD` 的 `Seq` 自增。
- 发送后等待同 `Seq` 的 `MSG_ACK`。
- 查询命令收到 ACK 后，再等待同 `Seq + CmdSet + CmdID` 的 `MSG_DATA`。
- 心跳周期 50~100 ms。
- ACK 超时或串口断开时，停止继续发送运动命令。
- 上位机不要发送底层设备原始帧。

## 13. 实施顺序

建议按以下顺序实现：

1. 协议层接管 `USART6`，完成帧解析、CRC、ACK 发送。
2. 实现 `SYS_PING`，确认上位机能收到 ACK。
3. 实现 `SYS_HEARTBEAT` 和心跳超时停止。
4. 实现 `SAFETY_ESTOP`、`SAFETY_SAFE_STOP`。
5. 实现 `CHASSIS_ENABLE`、`CHASSIS_STOP`、`CHASSIS_SET_MOTOR_RPM`、`CHASSIS_MOVE_MECANUM`。
6. 增加 `SENSOR_GET_OPS`、`SENSOR_GET_IMU` 查询。
7. 增加 `MSG_STATUS` 周期状态。
8. 后续按 `CMDSET_SENSOR` 规则接入其他传感器。

首版只追求简单、稳定、可控。扩展能力通过新增 `CmdID` 和 Payload 完成，不修改基础帧格式。
