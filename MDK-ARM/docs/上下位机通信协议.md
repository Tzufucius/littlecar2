# STM32F407 上下位机通信协议文档

## 1. 文档定位

本文档定义 Jetson Orin Nano 上位机和 STM32F407 下位机之间的串口通信协议。

```text
Jetson Orin Nano 上位机  <->  STM32F407 下位机
```

上位机可以调用哪些能力、这些能力如何映射到 STM32 内部函数，统一维护在：

```text
docs/上位机可调用能力清单.md
```

本文档只规定：

- 串口链路
- 帧格式
- 命令分类
- Payload 编码
- ACK、查询响应、状态上报
- 分阶段实现计划

设计原则：

```text
1. 协议尽量简单，方便 Jetson 侧组包和 STM32 侧解析。
2. 保留 CmdSet + CmdID，便于后续增加舵机、OPS、IMU、任务序列等能力。
3. 不把 ZDT、Emm_V5、OPS/WIT 中断回调等底层函数直接暴露给 Jetson。
4. 首期只实现能让小车稳定受控的最小命令集。
```

## 2. 化简结论

原草案中有 `FrameType`、`Flags`、`ACK`、`EVENT`、`STATUS`、`RESP`、`ERROR` 等较多概念，扩展性强，但首版实现成本偏高。

建议化简为：

```text
固定帧头 + 版本 + 消息类型 + 命令组 + 命令 ID + 序号 + 长度 + Payload + CRC16
```

主要化简点：

| 原设计 | 简化后 |
| --- | --- |
| `Flags` 标志位 | 删除，命令行为由 CmdSet/CmdID 固定定义 |
| `EVENT` 动作完成事件 | 首期删除，动作执行结果先通过 ACK 和周期 STATUS 表示 |
| `RESP` / `ERROR` 独立帧 | 合并到 `MSG_DATA` 和 `MSG_ACK` |
| 大量首期命令 | 收缩为心跳、急停、底盘、舵机、传感器查询、状态上报 |
| 复杂闭环命令 | 后续阶段再加，不进入首版 |

保留项：

- 二进制帧
- 双帧头 `0x5A 0xA5`
- `Length`
- `Seq`
- `CmdSet + CmdID`
- `CRC16-Modbus`
- 小端编码

## 3. 通信链路

上下位机主通信使用 `USART6`。

| Jetson | STM32F407 |
| --- | --- |
| TX | PC7 / USART6_RX |
| RX | PC6 / USART6_TX |
| GND | GND |

串口参数：

| 参数 | 建议值 |
| --- | --- |
| BaudRate | 首期 `115200`，稳定后可提高到 `460800` 或 `921600` |
| DataBits | `8` |
| StopBits | `1` |
| Parity | `None` |
| FlowControl | `None` |

STM32 接收建议：

```text
USART6_RX: DMA Circular + IDLE
USART6_TX: DMA Normal 或普通发送均可
```

首期建议优先保证 RX DMA 稳定。TX DMA 可用于后续高频状态上报。

## 4. 简化帧格式

### 4.1 帧结构

```text
Header1 Header2 Version MsgType CmdSet CmdID Seq_L Seq_H Length Payload CRC_L CRC_H
```

字段表：

| 偏移 | 字段 | 长度 | 类型 | 说明 |
| ---: | --- | ---: | --- | --- |
| 0 | Header1 | 1 | uint8_t | 固定 `0x5A` |
| 1 | Header2 | 1 | uint8_t | 固定 `0xA5` |
| 2 | Version | 1 | uint8_t | 当前 `0x01` |
| 3 | MsgType | 1 | uint8_t | 消息类型 |
| 4 | CmdSet | 1 | uint8_t | 命令组 |
| 5 | CmdID | 1 | uint8_t | 命令 ID |
| 6 | Seq_L | 1 | uint8_t | 序号低字节 |
| 7 | Seq_H | 1 | uint8_t | 序号高字节 |
| 8 | Length | 1 | uint8_t | Payload 长度，0~255 |
| 9... | Payload | N | bytes | 数据区 |
| 9+N | CRC_L | 1 | uint8_t | CRC16 低字节 |
| 10+N | CRC_H | 1 | uint8_t | CRC16 高字节 |

基础头长度：

```c
#define PROTO_HEAD_LEN 9
```

整帧长度：

```text
total_len = 9 + Length + 2
```

### 4.2 为什么保留 CRC16

虽然可以进一步简化为累加和校验，但上下位机之间会传运动控制、急停和传感器状态，错误帧不能被误执行。

因此首版仍保留 `CRC16-Modbus`：

| 项目 | 值 |
| --- | --- |
| 算法 | CRC16-Modbus |
| 初值 | `0xFFFF` |
| 多项式 | `0xA001` |
| 输出 | 低字节在前 |

CRC 计算范围：

```text
Version 到 Payload 最后一个字节
```

即不包含：

```text
Header1 Header2 CRC_L CRC_H
```

## 5. 消息类型

```c
typedef enum
{
    MSG_CMD    = 0x01,
    MSG_ACK    = 0x02,
    MSG_DATA   = 0x03,
    MSG_STATUS = 0x04
} ProtoMsgType_t;
```

| MsgType | 方向 | 用途 |
| ---: | --- | --- |
| `0x01` | Jetson -> STM32 | 控制命令或查询命令 |
| `0x02` | STM32 -> Jetson | 命令接收和执行结果 |
| `0x03` | STM32 -> Jetson | 查询响应数据 |
| `0x04` | STM32 -> Jetson | 周期状态上报 |

首期不单独设置 `EVENT`。如果命令是立即执行类，ACK 表示执行结果；如果命令是持续动作类，ACK 表示已接受，后续状态通过 `MSG_STATUS` 查看。

## 6. 命令组

```c
typedef enum
{
    CMDSET_SYSTEM  = 0x01,
    CMDSET_SAFETY  = 0x02,
    CMDSET_CHASSIS = 0x03,
    CMDSET_SERVO   = 0x04,
    CMDSET_SENSOR  = 0x05,
    CMDSET_CONFIG  = 0x06,
    CMDSET_DEBUG   = 0x7F
} ProtoCmdSet_t;
```

| CmdSet | 用途 |
| ---: | --- |
| `0x01` | 系统通信、心跳、模式 |
| `0x02` | 急停、安全停止 |
| `0x03` | 底盘控制 |
| `0x04` | 舵机、夹爪、机械臂 |
| `0x05` | OPS、IMU、电机状态等传感器数据 |
| `0x06` | 参数、预设点位、预设姿态 |
| `0x7F` | 调试命令 |

## 7. 通用返回码

ACK Payload 固定 4 字节：

```text
ack_seq_l ack_seq_h result detail
```

| 偏移 | 字段 | 类型 | 长度 | 说明 |
| ---: | --- | --- | ---: | --- |
| 0 | ack_seq | uint16_t | 2 | 被确认的 Jetson 命令序号 |
| 2 | result | uint8_t | 1 | 返回结果 |
| 3 | detail | uint8_t | 1 | 细节码，首期可填 0 |

`result`：

| 值 | 名称 | 含义 |
| ---: | --- | --- |
| `0x00` | OK | 执行成功或命令已接受 |
| `0x01` | BAD_CRC | CRC 错误 |
| `0x02` | BAD_LENGTH | Payload 长度错误 |
| `0x03` | BAD_PARAM | 参数错误 |
| `0x04` | BUSY | 系统忙 |
| `0x05` | DENIED | 当前状态不允许执行 |
| `0x06` | UNKNOWN_CMD | 未知命令 |
| `0x07` | FAULT | 系统故障 |

规则：

- 所有 `MSG_CMD` 都返回 `MSG_ACK`。
- 查询类命令先返回 ACK，再返回 `MSG_DATA`。
- CRC 错误时，如果无法确认 Seq，可以直接丢弃，不强制回复。

## 8. 数据编码规则

统一使用小端：

| 类型 | 编码 |
| --- | --- |
| `uint16_t` | 低字节在前 |
| `int16_t` | 低字节在前 |
| `uint32_t` | 低字节在前 |
| `int32_t` | 低字节在前 |

协议字段不直接使用 `float`，统一转为定点整数：

| 物理量 | 类型 | 单位 |
| --- | --- | --- |
| 电机转速 | `int16_t` | RPM |
| 舵机位置 | `int32_t` | 舵机原始位置 |
| 位置 | `int32_t` | mm |
| 角度 | `int32_t` | 0.01 deg |
| 角速度 | `int16_t` | 0.01 deg/s |
| 加速度 | `int16_t` | 0.001 g |
| 时间 | `uint32_t` | ms |

## 9. 首期命令定义

### 9.1 SYSTEM

| CmdID | 命令 | Payload | 说明 |
| ---: | --- | --- | --- |
| `0x01` | `SYS_PING` | 无 | 通信测试 |
| `0x02` | `SYS_HEARTBEAT` | `uint32_t jetson_time_ms` | 上位机心跳 |
| `0x03` | `SYS_SET_MODE` | `uint8_t mode` | 设置控制模式 |

`SYS_HEARTBEAT` 建议 50~100 ms 发送一次。STM32 超过 300 ms 没收到心跳，应停止底盘并进入安全状态。

### 9.2 SAFETY

| CmdID | 命令 | Payload | 说明 |
| ---: | --- | --- | --- |
| `0x01` | `SAFETY_ESTOP` | `uint8_t source` | 立即急停 |
| `0x02` | `SAFETY_SAFE_STOP` | `uint8_t mode` | 安全停止 |
| `0x03` | `SAFETY_CLEAR` | 无 | 清除可恢复安全状态 |

首期安全控制只要求能停止底盘。舵机和机械臂停止逻辑后续再接入。

### 9.3 CHASSIS

底盘命令首期直接对齐 `Core/Inc/chassis_motion.h` 已有能力。

| CmdID | 命令 | Payload 长度 | STM32 内部落点 |
| ---: | --- | ---: | --- |
| `0x01` | `CHASSIS_ENABLE` | 1 | `Chassis_Enable()` |
| `0x02` | `CHASSIS_STOP` | 1 | `Chassis_Stop()` / `Chassis_SmoothStop()` |
| `0x03` | `CHASSIS_SET_MOTOR_RPM` | 9 | `Chassis_SetMotorRPMEx()` |
| `0x04` | `CHASSIS_MOVE_MECANUM` | 7 | `Chassis_MoveMecanumEx()` |

`CHASSIS_ENABLE`：

| 偏移 | 字段 | 类型 | 说明 |
| ---: | --- | --- | --- |
| 0 | enable | uint8_t | `0` 失能，`1` 使能 |

`CHASSIS_STOP`：

| 偏移 | 字段 | 类型 | 说明 |
| ---: | --- | --- | --- |
| 0 | mode | uint8_t | `0` 平滑停止，`1` 立即停止 |

`CHASSIS_SET_MOTOR_RPM`：

| 偏移 | 字段 | 类型 | 单位 |
| ---: | --- | --- | --- |
| 0 | lf_rpm | int16_t | RPM |
| 2 | rf_rpm | int16_t | RPM |
| 4 | lr_rpm | int16_t | RPM |
| 6 | rr_rpm | int16_t | RPM |
| 8 | acc | uint8_t | Emm 加速度参数 |

`CHASSIS_MOVE_MECANUM`：

| 偏移 | 字段 | 类型 | 单位 |
| ---: | --- | --- | --- |
| 0 | forward_rpm | int16_t | RPM |
| 2 | strafe_rpm | int16_t | RPM |
| 4 | rotate_rpm | int16_t | RPM |
| 6 | acc | uint8_t | Emm 加速度参数 |

说明：

- `forward_rpm > 0` 表示前进。
- `strafe_rpm > 0` 表示右平移。
- `rotate_rpm > 0` 表示右旋转。
- 复杂的 `MoveRelative`、`GotoPose` 暂不进入首期。

### 9.4 SERVO

| CmdID | 命令 | Payload 长度 | STM32 内部落点 |
| ---: | --- | ---: | --- |
| `0x01` | `SERVO_SET_POSITION` | 9 | `BusServo_SetPositionEx()` |
| `0x02` | `SERVO_SET_GROUP` | 变长 | `BusServo_SendGroup()` |

`SERVO_SET_POSITION`：

| 偏移 | 字段 | 类型 | 说明 |
| ---: | --- | --- | --- |
| 0 | id | uint8_t | 舵机 ID |
| 1 | position | int32_t | 舵机原始位置 |
| 5 | acceleration | uint16_t | 加速度 |
| 7 | speed | uint16_t | 速度 |

`SERVO_SET_GROUP`：

```text
uint8_t count
重复 count 次：
    uint8_t id
    int32_t position
    uint16_t acceleration
    uint16_t speed
```

长度：

```text
Length = 1 + count * 9
```

首期不定义夹取、放置等复杂机械臂动作。后续通过 `CMDSET_SERVO` 增加 `ARM_ACTION` 即可。

## 10. 周期状态上报

STM32 可按 10 Hz 上报 `MSG_STATUS`。首期状态 Payload 控制在 32 字节以内。

建议首期 STATUS：

| 偏移 | 字段 | 类型 | 说明 |
| ---: | --- | --- | --- |
| 0 | stm32_time_ms | uint32_t | HAL tick |
| 4 | system_state | uint8_t | 系统状态 bit mask |
| 5 | fault_code | uint8_t | 故障码 |
| 6 | chassis_state | uint8_t | 底盘状态 |
| 7 | servo_state | uint8_t | 舵机状态 |
| 8 | ops_valid | uint8_t | OPS 是否有效 |
| 9 | imu_valid_mask | uint8_t | IMU 有效位 |
| 10 | pos_x_mm | int32_t | OPS X |
| 14 | pos_y_mm | int32_t | OPS Y |
| 18 | yaw_cdeg | int32_t | OPS 航向 |
| 22 | wz_cdeg_s | int16_t | Z 轴角速度 |

后续如果要加电机状态、电池、电流等内容，优先增加新的 `SENSOR_GET_*` 查询命令，不要把 STATUS 无限加长。

## 11. 解析与执行规则

STM32 接收规则：

```text
1. USART6_RX 使用 DMA + IDLE 接收字节流。
2. 协议层只负责找帧头、读 Length、校验 CRC、投递完整帧。
3. 中断或 DMA 回调中不直接控制电机和舵机。
4. 完整命令进入主循环或命令队列后再执行。
5. 每个 CmdID 必须检查 Length。
6. CRC 错误帧直接丢弃。
7. 未知命令返回 ACK_UNKNOWN_CMD。
```

Jetson 发送规则：

```text
1. 每条命令 Seq 自增。
2. 发送命令后等待对应 ack_seq 的 ACK。
3. 查询命令收到 ACK 后，再等待同 CmdSet/CmdID 的 MSG_DATA。
4. 心跳周期 50~100 ms。
5. 连接断开或 ACK 超时，上位机应停止继续发送运动命令。
```

## 12. 与底层设备协议的关系

上下位机协议不是张大头电机协议，也不是 OPS 或 WIT 原始协议。

```text
Jetson
  ↓ 上下位机协议
STM32 USART6 协议层
  ↓ 命令分发
chassis_motion / bus_servo / ops_sensor / wit_imu
  ↓
USART3 电机 / UART4 舵机 / UART5 OPS / USART2 IMU
```

Jetson 不应直接发送张大头、总线舵机、OPS、WIT 的原始帧。这样后续更换底层设备时，上位机协议不需要整体重写。

## 13. 分阶段实现计划

### 第一阶段：最小可控链路

目标：Jetson 能稳定连接 STM32，并控制底盘基础运动。

实现内容：

- `SYS_PING`
- `SYS_HEARTBEAT`
- `SAFETY_ESTOP`
- `SAFETY_SAFE_STOP`
- `CHASSIS_ENABLE`
- `CHASSIS_STOP`
- `CHASSIS_SET_MOTOR_RPM`
- `CHASSIS_MOVE_MECANUM`
- 通用 ACK

验收标准：

- Jetson 能收到 PING 的 ACK。
- 心跳正常时允许执行底盘命令。
- 心跳超时后 STM32 停止底盘。
- 急停命令能立即停止底盘。
- 底盘能按四轮 RPM 和麦克纳姆三自由度命令运动。

### 第二阶段：舵机与传感器查询

目标：上位机能控制舵机，并读取 OPS、IMU 数据。

实现内容：

- `SERVO_SET_POSITION`
- `SERVO_SET_GROUP`
- `SENSOR_GET_OPS`
- `SENSOR_GET_IMU`
- `MSG_DATA` 查询响应

验收标准：

- Jetson 能控制单个舵机位置。
- Jetson 能发送一组舵机目标。
- Jetson 能查询 OPS 位姿。
- Jetson 能查询 IMU 加速度、角速度和角度。

### 第三阶段：周期状态上报

目标：STM32 周期向 Jetson 上报系统和传感器简要状态。

实现内容：

- `MSG_STATUS`
- 10 Hz 状态上报
- 系统状态、故障码、底盘状态、舵机状态
- OPS 简要位姿
- IMU 有效位

验收标准：

- Jetson 能持续解析 STATUS。
- STATUS 不影响控制命令响应。
- 传感器失效时 valid 位能正确反映。

### 第四阶段：高级动作扩展

目标：在不破坏首版协议的基础上扩展复杂能力。

可扩展内容：

- `CHASSIS_MOVE_REL`
- `CHASSIS_GOTO_POSE`
- `ARM_ACTION`
- `ARM_TO_PRESET`
- `CONFIG_SET_STATION`
- `CONFIG_SET_ARM_PRESET`

扩展规则：

- 新增能力只增加 CmdID，不修改基础帧格式。
- 复杂动作可先 ACK 表示接受，再通过 STATUS 表示执行状态。
- 只有确实需要动作完成通知时，再考虑恢复独立 EVENT 帧。

## 14. 最终建议

首版协议不要追求一次覆盖全部能力。建议先实现“简单、稳定、可扩展”的最小集：

```text
帧格式固定
无 Flags
无 EVENT
所有命令都有 ACK
查询命令用 DATA 返回
周期信息用 STATUS 返回
新增功能靠 CmdSet + CmdID 扩展
```

这样 STM32 侧解析难度低，Jetson 侧组包也简单，后续增加舵机动作、传感器上报和高级导航时不会破坏已有通信基础。
