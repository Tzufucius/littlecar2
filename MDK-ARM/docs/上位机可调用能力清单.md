# 上位机可调用能力清单

## 1. 定位

本文档记录上位机可以调用的 STM32 能力。这里列的是稳定业务能力，不是 STM32 内部函数全集。

调用关系：

```text
上位机命令
  ↓
上下位机协议 CmdSet / CmdID
  ↓
STM32 命令分发层
  ↓
底盘 / 舵机 / 传感器模块
  ↓
具体设备协议
```

协议帧格式、ACK、CRC、状态上报规则见：

```text
docs/上下位机通信协议.md
```

## 2. 当前能力分组

| 分组 | CmdSet | 当前定位 | 首版是否实现 |
| --- | ---: | --- | --- |
| SYSTEM | `0x01` | 通信测试、心跳、模式 | 是 |
| SAFETY | `0x02` | 急停、安全停止、恢复 | 是 |
| CHASSIS | `0x03` | 底盘基础运动控制 | 是 |
| SERVO | `0x04` | 舵机、夹爪、机械臂 | 后续 |
| SENSOR | `0x05` | 传感器查询、状态上报 | 后续扩展 |
| CONFIG | `0x06` | 参数和预设 | 后续 |
| DEBUG | `0x7F` | 临时调试 | 按需 |

首版优先保证上位机能稳定控制底盘。传感器能力保留扩展位，按模块逐个接入。

## 3. SYSTEM

| 能力名称 | CmdID | 方向 | Payload | STM32 内部落点 | 状态 |
| --- | ---: | --- | --- | --- | --- |
| `System.Ping` | `0x01` | 上位机 -> STM32 | 无 | 协议层直接 ACK | 待实现 |
| `System.Heartbeat` | `0x02` | 上位机 -> STM32 | `uint32_t jetson_time_ms` | 心跳状态缓存 | 待实现 |
| `System.SetMode` | `0x03` | 上位机 -> STM32 | `uint8_t mode` | 系统状态机 | 待实现 |

说明：

- `System.Ping` 用于确认链路和协议解析可用。
- `System.Heartbeat` 用于判断上位机是否在线。
- 心跳超时后，STM32 应停止底盘并拒绝普通运动命令。

## 4. SAFETY

| 能力名称 | CmdID | 方向 | Payload | STM32 内部落点 | 状态 |
| --- | ---: | --- | --- | --- | --- |
| `Safety.EStop` | `0x01` | 上位机 -> STM32 | `uint8_t source` | `Chassis_Stop()` | 部分支撑 |
| `Safety.SafeStop` | `0x02` | 上位机 -> STM32 | `uint8_t mode` | `Chassis_SmoothStop()` / `Chassis_Stop()` | 部分支撑 |
| `Safety.Clear` | `0x03` | 上位机 -> STM32 | 无 | 安全状态清除 | 待实现 |

安全命令优先级高于普通底盘命令。首版只要求能可靠停止底盘，后续再扩展舵机、机械臂停止逻辑。

## 5. CHASSIS

当前底盘能力以 `Core/Inc/advance_chassis.h` 和 `Core/Inc/advance_world.h` 为边界。上位机不直接调用 `drive_emm_*`、`Emm_V5_*`、`Datou_*` 或 `ZDT_*`。

| 能力名称 | CmdID | 方向 | Payload | STM32 内部落点 | 状态 |
| --- | ---: | --- | --- | --- | --- |
| `Chassis.Enable` | `0x01` | 上位机 -> STM32 | `uint8_t enable` | `Chassis_Enable()` | 已有底层 |
| `Chassis.Stop` | `0x02` | 上位机 -> STM32 | `uint8_t mode` | `Chassis_Stop()` / `Chassis_SmoothStop()` | 已有底层 |
| `Chassis.SetMotorRPM` | `0x03` | 上位机 -> STM32 | `lf/rf/lr/rr_rpm`、`acc` | `Chassis_SetMotorRPMEx()` | 已有底层 |
| `Chassis.MoveMecanum` | `0x04` | 上位机 -> STM32 | `forward_rpm`、`strafe_rpm`、`rotate_rpm`、`acc` | `Chassis_MoveMecanumEx()` | 已有底层 |

首版只开放以上 4 个底盘命令。已有的 `Forward`、`Backward`、`StrafeLeft`、`RotateRight` 等函数保留为 STM32 内部便捷接口，不单独暴露为协议命令，避免命令表膨胀。

后续高级底盘能力预留：

| 能力名称 | 建议 CmdID | 说明 |
| --- | ---: | --- |
| `Chassis.MoveRelative` | `0x20` | 相对位移，需要位置闭环 |
| `Chassis.GotoPose` | `0x21` | 到达指定坐标，需要 OPS / IMU / 融合状态 |

`Chassis.GotoPose` 使用 `advance_world` 维护的 world 坐标系：`world +Y` 为小车初始车头方向，`world +X` 为小车初始右侧方向，`yaw=0` 朝 `world +Y`，yaw 逆时针为正。首版先完成 `advance_world` 位姿维护和调试输出，闭环执行再接入 `advance_chassis`。

## 6. SERVO

当前工程已有总线舵机设备层，位于 `Core/Inc/bus_servo.h` 和 `Core/Src/bus_servo.c`。它只负责发送舵机位置命令，不包含夹取、摆放等流程动作。

| 能力名称 | CmdID | 方向 | Payload | STM32 内部落点 | 状态 |
| --- | ---: | --- | --- | --- | --- |
| `Servo.SetPosition` | `0x01` | 上位机 -> STM32 | `id`、`position`、`acceleration`、`speed` | `BusServo_SetPositionEx()` | 已有底层 |
| `Servo.SetGroup` | `0x02` | 上位机 -> STM32 | `count` + 多个舵机目标 | `BusServo_SendGroup()` | 已有底层 |
| `Servo.GetLastStatus` | `0x03` | 上位机 -> STM32 | 无 | `BusServo_GetLastStatus()` | 已有底层 |

机械臂流程动作后续在业务层增加，不把机械结构标定值写进设备层。

预留能力：

| 能力名称 | 建议 CmdID | 说明 |
| --- | ---: | --- |
| `Arm.Stop` | `0x10` | 停止机械臂动作 |
| `Arm.Action` | `0x11` | 执行动作序列 |
| `Arm.ToPreset` | `0x12` | 运动到预设姿态 |

## 7. SENSOR

传感器能力统一使用 `CMDSET_SENSOR = 0x05`。新增传感器时只增加 `CmdID` 和 Payload，不修改基础帧格式。

| 能力名称 | CmdID | 方向 | Payload | STM32 内部落点 | 状态 |
| --- | ---: | --- | --- | --- | --- |
| `Sensor.GetOPS` | `0x01` | 上位机 -> STM32 | 无 | `OPS_GetPose()` | 已有底层 |
| `Sensor.GetIMU` | `0x02` | 上位机 -> STM32 | 无 | `WIT_GetData()` | 已有底层 |
| `Sensor.GetPoseFusion` | `0x03` | 上位机 -> STM32 | 无 | `AdvanceWorld_GetPose()` | 部分支撑 |
| `Sensor.GetMotorState` | `0x04` | 上位机 -> STM32 | 可选电机 ID | 电机状态缓存 | 待实现 |
| `Sensor.StatusReport` | `0x80` | STM32 -> 上位机 | 周期状态摘要 | 状态汇总层 | 待实现 |

### OPS 返回建议

| 字段 | 来源 | 协议单位 |
| --- | --- | --- |
| `zangle_cdeg` | `OPS_Pose_t.zangle_deg` | 0.01 deg |
| `xangle_cdeg` | `OPS_Pose_t.xangle_deg` | 0.01 deg |
| `yangle_cdeg` | `OPS_Pose_t.yangle_deg` | 0.01 deg |
| `pos_x_mm` | `OPS_Pose_t.pos_x_mm` | mm |
| `pos_y_mm` | `OPS_Pose_t.pos_y_mm` | mm |
| `wz_cdeg_s` | `OPS_Pose_t.w_z_dps` | 0.01 deg/s |
| `valid` | `OPS_Pose_t.valid` | 0 / 1 |

### IMU 返回建议

| 字段 | 来源 | 协议单位 |
| --- | --- | --- |
| `accel_x/y/z_mg` | `WIT_Data_t.accel_g` | 0.001 g |
| `gyro_x/y/z_cdeg_s` | `WIT_Data_t.gyro_dps` | 0.01 deg/s |
| `angle_x/y/z_cdeg` | `WIT_Data_t.angle_deg` | 0.01 deg |
| `valid_mask` | 三组 valid 标志 | bit mask |

### 新增传感器登记规则

新增传感器时，在本节补齐：

1. 能力名称，建议使用 `Sensor.GetXxx`。
2. `CmdID`，优先使用 `0x01~0x1F` 的查询区间。
3. 查询 Payload，如果无参数则写“无”。
4. 返回 Payload 字段、类型、单位、缩放倍数。
5. STM32 内部落点，例如 `Xxx_GetData()` 或状态缓存。
6. 是否进入 `Sensor.StatusReport` 摘要。

不建议把完整传感器原始帧透传给上位机。原始帧解析应留在 STM32 对应驱动模块中。

## 8. CONFIG 和 DEBUG

首版不开放配置类命令。后续确实需要参数下发时，再使用 `CMDSET_CONFIG = 0x06`。

调试命令使用 `CMDSET_DEBUG = 0x7F`，只用于临时联调，不作为正式业务入口。

| 能力名称 | 建议 CmdID | 说明 |
| --- | ---: | --- |
| `Config.SetChassisPreset` | `0x01` | 设置底盘预设速度、加速度 |
| `Config.SetStation` | `0x02` | 设置点位坐标 |
| `Config.SetArmPreset` | `0x03` | 设置机械臂预设姿态 |
| `Debug.RawMotorReadSpeed` | `0x01` | 临时读取电机速度 |
| `Debug.RawMotorReadPosition` | `0x02` | 临时读取电机位置 |

## 9. 不对上位机开放的内容

以下内容属于 STM32 内部实现，不列为上位机能力：

| 类型 | 示例 | 原因 |
| --- | --- | --- |
| 初始化函数 | `BusServo_Init()`、`OPS_Init()`、`WIT_Init()` | 上电生命周期内部执行 |
| 轮询函数 | `BusServo_Poll()`、`OPS_Poll()`、`WIT_Poll()` | 主循环内部维护状态 |
| 中断回调 | `OPS_OnByteReceived()`、`WIT_OnUartRxEvent()` | 串口接收链路内部使用 |
| 底层电机原生命令 | `Emm_V5_*`、`Datou_*`、`ZDT_*` | 避免上位机绑定设备协议 |
| 解析辅助函数 | 各设备协议 parser | 只服务底层协议解析 |

## 10. 维护规则

新增上位机能力时，必须同步维护：

1. 本文档中的能力名称、`CmdSet`、`CmdID`、方向和 Payload。
2. `docs/上下位机通信协议.md` 中对应 Payload 结构和执行规则。
3. STM32 命令分发逻辑。
4. 上位机组包和解析逻辑。

新增能力优先扩展 `CmdID`，不要修改基础帧格式。只有当前字段无法表达真实需求时，才考虑升级 `Version`。
