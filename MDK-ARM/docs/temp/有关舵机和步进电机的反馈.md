# 舵机与步进电机反馈现状

更新时间：2026-07-23。

## 驱动层反馈

- `drive_emm` 保留 USART3 DMA/IDLE 回包解析、反馈缓存和底盘反馈监督能力。
- 总线舵机仍只实现位置命令发送；反馈接口保留在驱动层，但当前不参与机械臂高层动作。

## 机械臂高层策略

`advance_arm` 是完全开环、固定顺序的阻塞模块：不读取步进电机位置反馈、不判断到位、不检查舵机反馈、不检查机械臂限位，也不维护软件零点、当前位置、目标位置或任务状态。

- 伸出、收回、下降、上升：直接发送位置命令后 `HAL_Delay(1000U)`。
- 夹爪打开、闭合：忽略发送结果后 `HAL_Delay(1000U)`。
- `ARM_PICK`：伸出、下降、闭合夹爪、上升、收回。
- `ARM_PLACE`：伸出、下降、打开夹爪、上升、收回。

Pick/Place 正常流程约阻塞 5000 ms。固定等待不代表电机已经实际到位。

## 上位机协议

| CmdID | 命令 | 当前行为 |
| ---: | --- | --- |
| `0x10` | `ARM_GRAB` | 仅接受 1 字节开合标志，同步阻塞 1000 ms |
| `0x12` | `ARM_PICK` | 仅接受空 Payload，同步执行固定抓取流程 |
| `0x13` | `ARM_PLACE` | 仅接受空 Payload，同步执行固定放置流程 |
| `0x14` | `ARM_ABORT` | 仅接受空 Payload，停止两步进轴 |
| `0x11`、`0x15`、`0x16` | `ARM_CONFIG`、`ARM_GET_STATUS`、`ARM_RESET_ZERO` | 返回 `ACK_UNKNOWN_CMD` |

机械臂动作开始前 STM32 会停止底盘并刷新当前控制者心跳；ACK 在阻塞动作结束后发送。`ACK_OK` 只表示阻塞函数返回，不代表机构实际到位。阻塞期间普通协议不能立即中断当前动作。

## 上板验收

1. 在无碰撞风险条件下确认升降轴和滑台轴方向。
2. 标定四个步进动作和夹爪动作的 1000 ms 等待是否足以完成动作。
3. 验证 Pick/Place 的固定线性顺序、约 5 秒阻塞时间和底盘预停。
