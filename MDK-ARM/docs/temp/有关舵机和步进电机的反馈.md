# 舵机与步进电机反馈现状

更新时间：2026-07-12。

## 已实现

- `drive_emm` 已完成 USART3 DMA/IDLE 回包解析，周期读取 `S_VEL`、`S_CPOS`、`S_PERR`、`S_FLAG`，并缓存实时速度、位置、位置误差、使能、堵转、故障与更新时间。
- 机械臂可通过 `drive_emm_MonitorMotor()` 将升降和伸缩轴加入反馈轮询。`drive_emm_IsMotorFeedbackHealthy()` 和 `drive_emm_IsMotorReached()` 统一判断反馈新鲜度、故障/堵转和位置容差。
- `advance_arm` 已实现非阻塞抓取/放置状态机。步进命令不会因为发送成功而视为完成，后续状态必须等待 ZDT 实际位置到达目标容差。
- 启动策略为人工置零：操作者在上电前将两步进轴放到机械零点；`ARM_CONFIG` 配置完成且两路新鲜反馈可用后，当前驱动器位置被定义为本次软件坐标零点。
- 通信超时、堵转、故障、停止或急停会使步进轴坐标失效。坐标失效后拒绝抓取/放置任务，需重新人工置零并重新配置。
- `SAFETY_ESTOP` 同时停止底盘与机械臂，并锁定后续普通控制命令。

## 舵机边界

总线舵机当前仅实现位置命令发送，已预留 `BusServo_Feedback_t`、`BusServo_GetFeedback()` 和 `BusServo_IsReached()` 接口。由于尚未接入实际舵机的回读帧协议，反馈有效位不会自动置位。

因此，抓取/放置状态机对夹爪暂按 `servo_wait_ms` 等待完成；该等待只用于舵机，不用于步进轴。获得实际型号、读位置指令和寄存器定义后，应在 UART4 接收层填充反馈缓存，并将夹爪阶段切换为 `BusServo_IsReached()` 闭环判断。

## 上位机命令

`CMDSET_SERVO = 0x04`：

| CmdID | 命令 | 作用 |
| ---: | --- | --- |
| `0x10` | `ARM_GRAB` | 保持兼容的单次夹爪位置命令 |
| `0x11` | `ARM_CONFIG` | 配置两步进轴、夹爪、方向、速度、加速度和位置容差 |
| `0x12` | `ARM_PICK` | 启动非阻塞抓取流程 |
| `0x13` | `ARM_PLACE` | 启动非阻塞放置流程 |
| `0x14` | `ARM_ABORT` | 停止机械臂并使坐标失效 |
| `0x15` | `ARM_GET_STATUS` | 返回任务、坐标有效性、当前位置和目标位置 |

精确字段定义见 [上下位机通信协议](../上下位机通信协议.md)。

## 上板验收

1. 在机构无碰撞风险的条件下人工将升降、伸缩轴置于机械零点。
2. 发送 `ARM_CONFIG`，确认 `ARM_GET_STATUS` 返回两个轴的坐标有效性均为 `VALID`。
3. 低速执行抓取和放置，确认每个步进阶段均由实际反馈到位后再进入下一阶段。
4. 拔掉任一轴反馈或制造堵转，确认任务进入 `FAULT` 且再次启动被拒绝。
5. 触发急停，确认两轴停止、坐标失效，恢复后必须重新人工置零和配置。
