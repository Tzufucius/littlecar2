# 舵机与步进电机反馈现状

更新时间：2026-07-23。

## 驱动层反馈

- `drive_emm` 保留 USART3 DMA/IDLE 回包解析、反馈缓存和底盘反馈监督能力。
- 总线舵机仍只实现位置命令发送；`BusServo_Feedback_t`、`BusServo_GetFeedback()` 和 `BusServo_IsReached()` 是预留接口，当前不参与高层动作判定。

## 机械臂高层策略

`advance_arm` 不再读取步进电机或舵机反馈，也不维护软件零点、目标位置、任务状态机或到位超时。

- 伸缩轴和升降轴动作：先检查目标方向对应的光电限位，再发送一次相对位置命令并阻塞等待 1000 ms。
- 夹爪动作：发送固定位置命令，发送成功后阻塞等待 500 ms。
- `ARM_PICK` 与 `ARM_PLACE` 按“伸出、下降、夹爪、上升、回收”顺序同步执行；任一步限位阻塞或夹爪发送失败时停止两步进轴。

该方案以可读性和现场标定为优先。动作完成由固定等待时间近似，不能据此证明实际电机已经到位。

## 上位机协议

| CmdID | 命令 | 当前行为 |
| ---: | --- | --- |
| `0x10` | `ARM_GRAB` | 仅接受 1 字节开合标志，同步执行 |
| `0x12` | `ARM_PICK` | 仅接受空 Payload，同步执行固定抓取流程 |
| `0x13` | `ARM_PLACE` | 仅接受空 Payload，同步执行固定放置流程 |
| `0x14` | `ARM_ABORT` | 仅接受空 Payload，停止两步进轴 |
| `0x11`、`0x15`、`0x16` | `ARM_CONFIG`、`ARM_GET_STATUS`、`ARM_RESET_ZERO` | 返回 `ACK_UNKNOWN_CMD` |

机械臂动作开始前 STM32 会停止底盘并刷新当前控制者心跳；ACK 会在阻塞动作结束后发送。上位机等待 Pick/Place ACK 时，应使用大于 4.5 秒的超时，并理解阻塞期间新停止命令不能被主循环立即处理。

## 上板验收

1. 以低速确认升降上/下、伸缩前/后四个方向及对应限位。
2. 标定伸出、回收、下降、上升的 1000 ms 等待是否足以完成动作。
3. 标定夹爪打开、闭合的 500 ms 等待是否足以完成动作。
4. 分别验证 Pick/Place 的线性顺序、限位阻塞后的两轴停止与底盘预停。
