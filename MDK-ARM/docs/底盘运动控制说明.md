# 底盘运动控制说明

本文档说明当前工程中麦克纳姆轮底盘高层运动接口的使用方式。

当前实现文件：

- 头文件：`Core/Inc/chassis_motion.h`
- 源文件：`Core/Src/chassis_motion.c`
- 底层电机接口：`Core/Inc/Emm_V5.h`、`Core/Src/Emm_V5.c`

## 1. 模块定位

`chassis_motion` 是底盘运动控制层，不直接处理串口协议细节。

它负责把“前进、后退、左平移、右平移、原地旋转、差速转向”等车体动作，转换成四个 ZDT 电机的速度命令。

底层发送仍由 `Emm_V5` 完成：

```text
用户 / main / 后续 Jetson 协议
    ↓
chassis_motion
    ↓
Emm_V5 多电机命令
    ↓
USART3
    ↓
ZDT 电机驱动器
```

## 2. 电机 ID 配置

四个电机 ID 在 `Core/Inc/chassis_motion.h` 中配置：

```c
#define CHASSIS_MOTOR_LF_ID ((uint8_t)1U)
#define CHASSIS_MOTOR_RF_ID ((uint8_t)2U)
#define CHASSIS_MOTOR_LR_ID ((uint8_t)3U)
#define CHASSIS_MOTOR_RR_ID ((uint8_t)4U)
```

含义：

| 宏 | 位置 |
| --- | --- |
| `CHASSIS_MOTOR_LF_ID` | 左前轮 |
| `CHASSIS_MOTOR_RF_ID` | 右前轮 |
| `CHASSIS_MOTOR_LR_ID` | 左后轮 |
| `CHASSIS_MOTOR_RR_ID` | 右后轮 |

如果实际电机 ID 与默认值不同，只需要修改这些宏。

## 3. 电机方向修正

底盘层约定：轮子 RPM 为正时，对小车产生向前运动贡献。

如果某个电机的实际正转方向与该约定相反，在 `chassis_motion.h` 中将对应 `SIGN` 改为 `-1`：

```c
#define CHASSIS_MOTOR_LF_SIGN (1)
#define CHASSIS_MOTOR_RF_SIGN (1)
#define CHASSIS_MOTOR_LR_SIGN (1)
#define CHASSIS_MOTOR_RR_SIGN (1)
```

调试建议：

1. 先低速调用 `Chassis_Forward(50)`。
2. 观察四个轮子是否都推动小车前进。
3. 哪个轮子方向相反，就把对应 `CHASSIS_MOTOR_*_SIGN` 改为 `-1`。

## 4. 平滑移动预设参数

各运动动作的默认速度和加速度在 `chassis_motion.h` 中集中配置：

```c
#define CHASSIS_FORWARD_PRESET_RPM ((uint16_t)100U)
#define CHASSIS_FORWARD_PRESET_ACC ((uint8_t)10U)

#define CHASSIS_BACKWARD_PRESET_RPM ((uint16_t)100U)
#define CHASSIS_BACKWARD_PRESET_ACC ((uint8_t)10U)

#define CHASSIS_STRAFE_PRESET_RPM ((uint16_t)100U)
#define CHASSIS_STRAFE_PRESET_ACC ((uint8_t)10U)

#define CHASSIS_ROTATE_PRESET_RPM ((uint16_t)80U)
#define CHASSIS_ROTATE_PRESET_ACC ((uint8_t)8U)
```

参数含义：

| 参数 | 含义 |
| --- | --- |
| `RPM` | 目标转速，单位 RPM |
| `ACC` | Emm 速度模式加速度参数 |

`ACC` 越大，响应越快；`ACC` 越小，启动、变速、停止越平滑。Emm 协议中 `ACC=0` 表示直接启动，不适合平滑移动。

## 5. 运动接口

### 5.1 使能与停止

```c
Chassis_Enable(true);      // 使能四个底盘电机
Chassis_Enable(false);     // 失能四个底盘电机
Chassis_Stop();            // 立即停止
Chassis_SmoothStop(8);     // 按加速度参数平滑降到 0RPM
```

### 5.2 固定动作

```c
Chassis_Forward(100);       // 前进
Chassis_Backward(100);      // 后退
Chassis_StrafeLeft(100);    // 左平移
Chassis_StrafeRight(100);   // 右平移
Chassis_RotateLeft(80);     // 左原地旋转
Chassis_RotateRight(80);    // 右原地旋转
```

这些接口使用 `CHASSIS_DEFAULT_ACC` 作为加速度。

如果需要单次指定加速度，使用 `Ex` 接口：

```c
Chassis_ForwardEx(100, 8);
Chassis_StrafeLeftEx(80, 6);
Chassis_RotateRightEx(60, 5);
```

如果希望使用头文件中的预设参数，使用 `Preset` 接口：

```c
Chassis_ForwardPreset();
Chassis_BackwardPreset();
Chassis_StrafeLeftPreset();
Chassis_StrafeRightPreset();
Chassis_RotateLeftPreset();
Chassis_RotateRightPreset();
```

## 6. 轮速分配

当前四轮顺序为：

```text
LF  RF
LR  RR
```

固定动作的轮速分配如下：

| 动作 | LF | RF | LR | RR |
| --- | --- | --- | --- | --- |
| 前进 | + | + | + | + |
| 后退 | - | - | - | - |
| 左平移 | - | + | + | - |
| 右平移 | + | - | - | + |
| 左原地旋转 | - | + | - | + |
| 右原地旋转 | + | - | + | - |

如果车辆运动方向与表格不一致，优先检查：

1. 电机 ID 是否对应实际车轮位置。
2. `CHASSIS_MOTOR_*_SIGN` 是否需要反向。
3. 麦克纳姆轮安装方向是否左右/前后装反。

## 7. 差速转向

```c
Chassis_DifferentialTurn(left_rpm, right_rpm);
Chassis_DifferentialTurnEx(left_rpm, right_rpm, acc);
Chassis_DifferentialTurnPreset();
```

含义：

- `left_rpm` 控制左前轮和左后轮。
- `right_rpm` 控制右前轮和右后轮。
- 左右速度相等时直行。
- 左右速度不同但同号时转弯。
- 左右速度反号时接近原地旋转。

示例：

```c
Chassis_DifferentialTurnEx(120, 60, 8);   // 前进并向右转
Chassis_DifferentialTurnEx(60, 120, 8);   // 前进并向左转
```

## 8. 通用麦克纳姆轮运动

```c
Chassis_MoveMecanum(forward_rpm, strafe_rpm, wz_ccw_rpm);
Chassis_MoveMecanumEx(forward_rpm, strafe_rpm, wz_ccw_rpm, acc);
Chassis_MoveMecanumPreset();
```

参数约定：

| 参数 | 正数 | 负数 |
| --- | --- | --- |
| `forward_rpm` | 前进 | 后退 |
| `strafe_rpm` | 右平移 | 左平移 |
| `wz_ccw_rpm` | 逆时针旋转 | 顺时针旋转 |

轮速合成公式：

```text
LF = forward + strafe - wz_ccw
RF = forward - strafe + wz_ccw
LR = forward - strafe - wz_ccw
RR = forward + strafe + wz_ccw
```

示例：

```c
Chassis_MoveMecanumEx(100, 0, 0, 8);    // 平滑前进
Chassis_MoveMecanumEx(0, -80, 0, 8);    // 平滑左平移
Chassis_MoveMecanumEx(80, 0, 30, 8);    // 前进同时逆时针旋转
```

## 9. 使用注意

- 当前模块基于 `Emm_V5`，不是 `zdt_stepper`。
- 当前默认使用 `USART3`，由 `Emm_V5.h` 中的 `EMM_V5_UART_HANDLE` 决定。
- 当前通过 `Emm_V5_MMCL_*` 装载四电机命令，再用广播地址 `0x00` 发送多电机命令和同步启动命令。
- `CHASSIS_MAX_RPM` 默认限制为 `3000RPM`，避免速度超出 Emm 固件常用安全范围。
- 上电调试时建议先用 `50RPM` 以下低速测试，确认 ID 和方向后再提高速度。

## 10. 物理速度接口与方向配置

`advance_chassis` 现在新增物理速度接口：

```c
Chassis_SetBodyVelocity(vx_right_mm_s, vy_forward_mm_s, wz_ccw_deg_s);
Chassis_SetBodyVelocityEx(vx_right_mm_s, vy_forward_mm_s, wz_ccw_deg_s, acc);
```

方向约定：

| 参数 | 正方向 |
| --- | --- |
| `vx_right_mm_s` | 车体向右 |
| `vy_forward_mm_s` | 车体向前 |
| `wz_ccw_deg_s` | 俯视逆时针旋转 |

如果实车某个方向相反，不要改控制代码，优先调整 `Core/Inc/advance_chassis.h`：

```c
#define CHASSIS_BODY_X_SIGN (1)
#define CHASSIS_BODY_Y_SIGN (1)
#define CHASSIS_BODY_WZ_SIGN (1)
```

物理参数也集中在同一头文件中：

```c
#define CHASSIS_WHEEL_RADIUS_MM
#define CHASSIS_HALF_LENGTH_MM
#define CHASSIS_HALF_WIDTH_MM
#define CHASSIS_MOTOR_GEAR_RATIO
```

这些值必须按实车轮径、前后轮中心距、左右轮中心距和减速比重新标定。当前麦轮合成后的四轮 RPM 会按最大绝对值整体缩放，避免单轮截断破坏运动方向。

## 11. 世界速度与 GotoPose

`advance_motion` 位于 world 坐标运动控制层，当前提供两个层级：

- `AdvanceMotion_SetWorldVelocityEx(vx_world_mm_s, vy_world_mm_s, wz_ccw_deg_s, acc)`：直接发送 world 速度，内部转换为车体速度后调用底盘物理速度接口。
- `AdvanceMotion_GotoPoseEx(const WorldGoalPose2D_t *goal, uint8_t acc)`：启动异步目标点状态机。

`GotoPose` 不阻塞调用方。主循环需要主动调用：

```c
AdvanceMotion_Poll();
```

`main.c` 已接入 `AdvanceMotion_Poll()`；任何 RPM、麦轮、车体速度或世界速度新命令都会取消正在运行的 GotoPose，防止多个控制源同时写底盘。

目标点结构使用 `WorldGoalPose2D_t`：

| 字段 | 单位 | 说明 |
| --- | --- | --- |
| `x_mm` | mm | world 目标 X |
| `y_mm` | mm | world 目标 Y |
| `yaw_deg` | deg | 目标 yaw，俯视逆时针为正 |
| `vmax_mm_s` | mm/s | 平移速度上限；小于等于 0 使用默认值 |
| `wmax_deg_s` | deg/s | 旋转速度上限；小于等于 0 使用默认值 |
| `timeout_ms` | ms | 目标超时；为 0 表示不启用目标超时 |
| `goal_flags` | bit mask | bit0 启用 yaw 控制 |

控制逻辑：

1. 读取 `advance_world` 当前位姿，要求 world 原点已建立且位姿未超过 `ADVANCE_MOTION_POSE_TIMEOUT_MS`。
2. 计算 `target - current` 得到 world x/y 误差。
3. 使用 `ADVANCE_MOTION_KP_POS` 生成 world 平移速度，并按二维向量模长限幅到 `vmax_mm_s`。
4. 若 `goal_flags` 启用 `ADVANCE_MOTION_GOAL_USE_YAW`，对 yaw 误差做 wrap，再用 `ADVANCE_MOTION_KP_YAW` 生成 `wz_ccw_deg_s` 并限幅。
5. 调用 `Chassis_SetBodyVelocityEx()` 输出到底盘。
6. 位置误差小于 `ADVANCE_MOTION_POS_TOLERANCE_MM`，且启用 yaw 时 yaw 误差小于 `ADVANCE_MOTION_YAW_TOLERANCE_DEG`，连续保持 `ADVANCE_MOTION_ARRIVE_HOLD_MS` 后进入 `ARRIVED`。

失败与终止：

- world 原点未建立：进入 `NO_ORIGIN` 并平滑停车。
- 位姿无效或超时：进入 `NO_POSE` 并平滑停车。
- 超过目标 `timeout_ms`：进入 `TIMEOUT` 并平滑停车。
- 调用 `AdvanceMotion_Cancel()`：进入 `CANCELED` 并平滑停车。
- 直接调用 world 速度接口时，会取消正在运行的 GotoPose，避免两个控制源同时输出。

实车验证建议：

1. 先确认单轮 ID、`CHASSIS_MOTOR_*_SIGN` 和 `CHASSIS_BODY_*_SIGN`。
2. 调用 `AdvanceWorld_ResetOrigin()` 建立 world 原点。
3. 低速测试 `GotoPose(300, 0)` 和 `GotoPose(0, 300)`。
4. 再置位 `ADVANCE_MOTION_GOAL_USE_YAW` 验证小角度 yaw 目标。
5. 用 `CHASSIS_CANCEL_GOAL` 和 `CHASSIS_GET_MOTION_STATUS` 验证取消与状态查询。
