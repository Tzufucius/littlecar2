# 下位机闭环与安全修复说明

## 1. 初始化与调度

初始化顺序为：`OPS_Init/WIT_Init`、`CarPose_Init`、`AdvanceWorld_Init`、`AdvanceMotion_Init`、`drive_emm_Init`。主循环不再阻塞等待固定时间；OPS 位姿有效后，`AdvanceWorld_ResetOrigin()` 会自动建立原点。

TIM6 每 1 ms 只置位任务标志；主循环在无任务时执行 `__WFI()`，在有任务时按以下顺序延后执行：

```text
drive_emm_Poll
-> OPS_Poll / WIT_Poll
-> AdvanceWorld_Poll
-> HostRx_Poll
-> AdvanceMotion_Poll
```

`AdvanceMotion_Poll()` 仅在 `GotoPose` 处于 `RUNNING` 时输出速度，控制周期由 `ADVANCE_MOTION_CONTROL_PERIOD_MS` 配置。

调度周期为：协议 `2 ms`，world 位姿与安全检查 `10 ms`，运动控制与电机轮询 `20 ms`，状态灯 `500 ms`，原点重试 `1000 ms`。业务函数不直接运行在 TIM6 中断中。

## 2. 停止和控制权

急停、心跳超时、普通停止、取消目标和底盘失能都先取消活动目标点。急停与心跳超时使用 `Chassis_Stop()`；普通停止使用 `Chassis_SmoothStop()`。

底盘控制源互斥：四轮 RPM、麦轮 RPM、车体速度、世界速度和 `GotoPose` 不能同时持有控制权。手动命令会取消 `GotoPose`，避免状态机在下一周期覆盖新命令。

## 3. GotoPose 到达条件

进入位置和航向容差后，控制器立即发送零速度，并从此时开始计时。只有在 `ADVANCE_MOTION_ARRIVE_HOLD_MS` 内持续满足容差，状态才变为 `ARRIVED`。离开容差会清除保持计时并恢复闭环控制。

目标输入由 `advance_motion.h` 的以下配置限制：

- `ADVANCE_MOTION_WORLD_X/Y_MIN/MAX_MM`：场地可达范围。
- `ADVANCE_MOTION_MAX_VMAX_MM_S`、`ADVANCE_MOTION_MAX_WMAX_DEG_S`：最大目标速度。
- `ADVANCE_MOTION_MAX_TIMEOUT_MS`：最长目标超时。

## 4. 坐标和传感器配置

统一约定为 `+X 向右`、`+Y 向前`、`+WZ 逆时针`。`Chassis_MoveMecanumEx()` 的第三参数已改为 `wz_ccw_rpm`，与物理速度接口的旋转符号一致。

在 `advance_world.h` 中完成 OPS/WIT 安装标定：轴交换、轴反向、航向偏角以及 OPS 到底盘旋转中心的 X/Y 偏移。修改这些参数后必须依次低速验证右移、前进、逆时针旋转和 world 速度方向。

OPS 接收层会拒绝 `NaN/Inf`、量程超限及位置突变帧。实际场地与最高速度变化后，应重新标定 `OPS_MAX_*` 门限。

## 5. 电机通信监督

`drive_emm` 为 USART3 增加 DMA 发送队列，发送帧在复制进队列后才清空多电机装载缓存。`drive_emm_Poll()` 轮询每个底盘电机的实时转速、位置、位置误差和状态位。

任何发送超时、队列错误、反馈超时、堵转或掉电标志都会使 `drive_emm_IsChassisFeedbackHealthy()` 返回失败；若底盘存在非零运动命令，主循环会取消目标并立即停车。

默认会向四个底盘驱动器写入 `DRIVE_EMM_HEARTBEAT_PROTECT_MS = 500 ms` 的心跳保护。该参数为持久化配置，上板前应确认驱动器为支持该命令的 Emm 固件；不支持时将 `DRIVE_EMM_ENABLE_HEARTBEAT_PROTECTION` 改为 `0`，并保留 STM32 侧反馈超时保护。

## 6. 协议与中断边界

USART1/USART6 的 DMA 回调只负责接收和协议帧入队。`HostProtocol_Poll()` 在主循环调度任务中执行，ACK 和状态数据通过 UART 中断发送队列异步发送，不使用阻塞式 `HAL_UART_Transmit()`。

USART3 正常 DMA/IDLE 回调只调用 `drive_emm_OnUartRxEvent()`；`drive_emm_OnUartError()` 只在 UART 错误回调中调用。TIM6、UART 和 DMA 的 NVIC 优先级由 CubeMX 配置文件统一维护。

## 7. 上板验收顺序

1. 断开负载或抬起车轮，确认四个电机 ID、方向和使能状态。
2. 确认 USART3 可收到 `0x35`、`0x36`、`0x37`、`0x3A` 回包，再允许底盘运动。
3. 低速验证车体右移、前进、逆时针旋转与坐标约定一致。
4. 静止等待 OPS 有效，检查自动原点建立。
5. 低速验证 `GotoPose(300, 0)`、`GotoPose(0, 300)`、到达后零速度和 `ARRIVED`。
6. 验证急停、取消目标、心跳超时、拔掉任一电机反馈后的停车行为。
7. 用逻辑分析仪或 GPIO 翻转测量确认 TIM6 中断与主循环调度不会超过 1 ms 基准周期。
